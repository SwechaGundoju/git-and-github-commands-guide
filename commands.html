Here are some commonly used Git commands

Basic Commands

git init: Initializes a new Git repository.
git clone repository_url: Clones an existing repository.
git status: Shows the current status of the repository.

Basic Setup & Configuration

git config --global user.name "Your Name": Sets the global username for Git.
git config --global user.email "your.email@example.com": Sets the global email for Git.
git config --list: Lists all configured Git settings.

Working with Changes

git status: Displays the status of changes in the working directory.
git add filename: Stages a file for commit.
git commit -m "message": Commits the staged changes with a message.
git diff: Shows the differences between changes in the working directory and the index.
git reset filename: Unstages a file (keeps changes in the working directory).

  • git reset --hard:All changes in the working directory and staging area are lost. This is a destructive operation, so be careful when using it.
  • git reset --soft:Useful when you want to completely discard changes, both staged and unstaged, and reset everything to a clean state.

Branching and Merging

git branch:Lists all branches.
git branch branch_name:Creates a new branch.
git checkout branch_name: Switches to a different branch.
git checkout -b branch_name: Creates and switches to a new branch.
git merge branch_name: Merges changes from another branch into the current branch.
